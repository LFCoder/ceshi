
一.创建版本库
    1.创建一个空目录用于存放版本的所有文件(即该目录为工作区)
    2.git init 命令就会在当前目录下创建了版本库(一个隐藏文件.git)
    3.git add 文件名  用于想版本库中添加文件(其实是向暂缓区中添加文件)
    4.git commit 提交文件(把暂缓区中的内容提交到版本库中)
二.时光机穿梭
   1.git status 用来查看git版本库中的状态,例如是否有更改却没有体交得文件(查看工作区->缓存区->版本库中的区别)
   2.git diff   用来查看具体有哪些内容改变了 (工作区和缓存区中的不同)

三.工作区和暂缓区
   工作区: 就是git版本库所在目录里面所有的文件都是在工作区!
   缓存区: 在版本库中会存有一个缓存区,缓存区存放着上次 git add后所有的内容.

四.管理修改
    (1).git checkout —- <file> 用于撤销在工作区的变化(工作区与缓存区的相比较,工作区的变化:checkout检出的意思就是相当把缓存区的文件检出覆盖工作区内容)
    (2).git reset HEAD <file> 用于撤销缓存区中的变化(缓存区与版本库相比较,撤销掉缓存区中与版本库中的变化)
